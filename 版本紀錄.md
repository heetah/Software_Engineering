# 版本紀錄
## 目前版本 0.1.0 (2025.11.16)
### 內容:
1. 修改templates.js ， 將所有 Agent 共享的模板集中管理

2. 建立第一版的verifier agent ， 其主要功能:  
*  讀取 architecture.json
*  使用 templates.js（Verified 區塊）生成 test-plan.json
*  輸出檔案：/data/sessions/<sessionId>/test-plan.json

3. 建立第一版的tester agent ， 
*  讀取 test-plan.json
*  為每個 testFile 呼叫 LLM → 產生可執行 Jest 測試碼
*  將測試碼寫入 generated-tests/
*  執行 Jest: npx jest --json --outputFile jest-report.json
*  解析 jest-report.json
*  輸出: 1. test-report.json 2. error-report.json
* （可選）如果有錯誤，用 LLM 產生錯誤原因（suggestedCause）

### 目前遇到的問題:
1. 資料結構不符預期
  * Architect Agent 產生的是單一 session JSON 檔（如 data/sessions/<sessionId>.
    json），內容並沒有 modules 或 architecture.modules 欄位。
  * Verifier Agent 預期要有 modules（或 architecture.modules），否則無法產生
    test-plan.json。

2. 流程斷鏈
  * 因為缺少 modules，Verifier Agent 會報錯「session 檔未包含可用的 modules（期待 modules  或 architecture.modules）」。
  * 不確定什麼是 module，也不確定如何讓流程順利進行。

3. session Id 不是一個資料夾，可能會出現流程錯誤???

### 解決方案
1. architect agent 需要調整輸出格式:
  * 讓 Architect Agent 產生標準的 architecture.json
    * 路徑：data/sessions/<sessionId>/architecture.json
    * 格式需包含 modules 陣列，每個 module 需有 apis（API/函式）描述。

2. 路徑資料夾需要做更動:
  * 需要找到對應session Id 的資料夾
  * 若無，需要創建一個對應的資料夾，並將輸出改為此資料夾下方
